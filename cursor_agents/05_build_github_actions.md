# Step 5: Build GitHub Actions workflow templates

**Prerequisites:** Steps 1–2 (invoke.py must exist)
**Output:** `templates/brain_listen.yml`, `templates/experiment_resume.yml`
**Test:** Manual test with a test GitHub repo (see instructions below)

---

## What to build

Two GitHub Actions workflow files that are copied into the experiment repo's `.github/workflows/` during `autoPhD connect`.

---

## File to create: `templates/brain_listen.yml`

This action fires when the adapter pushes health/completion data. It checks out the brain repo and calls `brain/invoke.py`.

```yaml
# .github/workflows/brain_listen.yml
# Generated by AutoPhD brain/connect.py — do not edit manually
# Fires when the autoPhD_adapter.py commits health or completion data

name: AutoPhD Brain Listen

on:
  push:
    branches:
      - 'exp/**'
    paths:
      - '.autoPhD/heartbeat.json'
      - '.autoPhD/completion_flag.json'

jobs:
  invoke-brain:
    runs-on: ubuntu-latest
    # Prevent concurrent runs on the same branch — queue them
    concurrency:
      group: autophd-${{ github.ref }}
      cancel-in-progress: false

    steps:
      - name: Checkout experiment repo
        uses: actions/checkout@v4
        with:
          fetch-depth: 0   # needed for git history
          token: ${{ secrets.GITHUB_TOKEN }}

      - name: Configure git
        run: |
          git config user.name "AutoPhD Brain"
          git config user.email "autophd@noreply.github.com"

      - name: Checkout brain repo
        uses: actions/checkout@v4
        with:
          repository: {{BRAIN_REPO}}
          path: _autophd_brain
          token: ${{ secrets.AUTORESEARCH_READ_TOKEN || secrets.GITHUB_TOKEN }}

      - name: Set up Python 3.11
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
          cache: 'pip'
          cache-dependency-path: '_autophd_brain/requirements.txt'

      - name: Install brain dependencies
        run: pip install -r _autophd_brain/requirements.txt

      - name: Check kill switch
        id: kill_check
        run: |
          if [ -f ".autoPhD/interrupt.json" ]; then
            KILL=$(python3 -c "import json; d=json.load(open('.autoPhD/interrupt.json')); print(d.get('kill_permanent', False))")
            echo "kill_permanent=$KILL" >> $GITHUB_OUTPUT
          else
            echo "kill_permanent=False" >> $GITHUB_OUTPUT
          fi

      - name: Invoke brain
        if: steps.kill_check.outputs.kill_permanent != 'True'
        run: |
          python _autophd_brain/brain/invoke.py
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          EVENT_COMMIT_MSG: ${{ github.event.head_commit.message }}
          EXPERIMENT_BRANCH: ${{ github.ref_name }}
          EXPERIMENT_REPO_PATH: ${{ github.workspace }}
          BRAIN_REPO_PATH: ${{ github.workspace }}/_autophd_brain

      - name: Push brain outputs
        if: steps.kill_check.outputs.kill_permanent != 'True'
        run: |
          git add .autoPhD/
          if git diff --staged --quiet; then
            echo "No changes to commit"
          else
            git commit -m "brain: agent outputs for ${{ github.event.head_commit.message }}"
            git push origin ${{ github.ref_name }}
          fi
```

## File to create: `templates/experiment_resume.yml`

This action fires when the brain pushes back (interrupt, config update, pending approval). It does NOT restart the experiment — it creates a notification and ensures the branch is up to date.

```yaml
# .github/workflows/experiment_resume.yml
# Generated by AutoPhD brain/connect.py — do not edit manually
# Fires when the brain pushes intervention files to the experiment branch

name: AutoPhD Experiment Resume

on:
  push:
    branches:
      - 'exp/**'
    paths:
      - '.autoPhD/interrupt.json'
      - '.autoPhD/PENDING_APPROVAL.md'
      - '.autoPhD/NOTIFICATION.md'
      - '.autoPhD/config.json'

jobs:
  notify:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout experiment repo
        uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}

      - name: Log brain action
        run: |
          echo "Brain action detected at $(date)"
          echo "Commit: ${{ github.event.head_commit.message }}"
          echo "Files changed: ${{ join(github.event.commits[0].modified, ', ') }}"

      - name: Check for PENDING_APPROVAL
        run: |
          if [ -f ".autoPhD/PENDING_APPROVAL.md" ]; then
            echo "::warning::Human approval required. See .autoPhD/PENDING_APPROVAL.md"
          fi

      - name: Check for NOTIFICATION
        run: |
          if [ -f ".autoPhD/NOTIFICATION.md" ]; then
            echo "::notice::AutoPhD notification. See .autoPhD/NOTIFICATION.md"
            cat .autoPhD/NOTIFICATION.md
          fi

      - name: Check for interrupt
        run: |
          if [ -f ".autoPhD/interrupt.json" ]; then
            REASON=$(python3 -c "import json; d=json.load(open('.autoPhD/interrupt.json')); print(d.get('reason', 'unknown'))")
            KILL=$(python3 -c "import json; d=json.load(open('.autoPhD/interrupt.json')); print(d.get('kill_permanent', False))")
            echo "::warning::Interrupt signal set. Reason: $REASON | Kill permanent: $KILL"
          fi
```

## Manual test procedure

Since GitHub Actions cannot be unit-tested locally, use this procedure:

### Setup test repo

```bash
# Create a test experiment repo
mkdir test-exp-repo && cd test-exp-repo
git init
git remote add origin https://github.com/{you}/test-exp-repo.git
mkdir -p .autoPhD .github/workflows

# Copy the generated workflow files
cp /path/to/templates/brain_listen.yml .github/workflows/
cp /path/to/templates/experiment_resume.yml .github/workflows/
# Replace {{BRAIN_REPO}} with your actual brain repo

git add .
git commit -m "initial setup"
git push -u origin main

# Create experiment branch
git checkout -b exp/test-H001-20260317-143022
git push -u origin exp/test-H001-20260317-143022
```

### Set GitHub secrets

```bash
gh secret set ANTHROPIC_API_KEY --repo {you}/test-exp-repo --body "$ANTHROPIC_API_KEY"
gh secret set GEMINI_API_KEY --repo {you}/test-exp-repo --body "$GEMINI_API_KEY"
# AUTORESEARCH_READ_TOKEN only needed if brain repo is private
```

### Test brain_listen.yml

```bash
# Simulate adapter push
echo '{"health": "yellow", "iteration": 50}' > .autoPhD/heartbeat.json
git add .autoPhD/heartbeat.json
git commit -m "health: YELLOW iter=50/200 plateau_detected"
git push origin exp/test-H001-20260317-143022

# Check: GitHub Actions tab should show the workflow running
# Check: After run, .autoPhD/monitor_report.md should appear (if invoke.py works)
```

### Test kill switch

```bash
echo '{"interrupt": true, "reason": "test", "kill_permanent": true}' > .autoPhD/interrupt.json
git add .autoPhD/interrupt.json
git commit -m "brain: interrupt reason=test"
git push

# Then push a heartbeat — brain_listen.yml should run but kill_check should stop invoke.py
echo '{"health": "red"}' > .autoPhD/heartbeat.json
git add .autoPhD/heartbeat.json
git commit -m "health: RED iter=60/200 nan_detected"
git push
# Check: brain_listen.yml fires but invoke.py is NOT called (kill_permanent=True)
```
